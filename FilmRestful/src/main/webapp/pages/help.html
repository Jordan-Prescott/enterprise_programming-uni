NOTE: Format

On the home, add film, and update page there is a format selection near the top right of each page. In the selection you have JSON, XML, and TEXT this is the how the data will be formatted/ requested in each action on that page. For example if you are adding a page and select XML this will format your film in XML when sending to the API. If you are selected TEXT when searching films on the main page the films returned from the API will be in this format, parsed and then displayed in the films table.

The main page is where all films in the database are presented in a table and where you access all features of this application. This is a simple page with a navigation bar across the top and the films table in the body.

Notifications

The notification bell is where all actions and notifications are shown. For example when you delete a film the message back from the API will be displayed here indicating if this was a success or if there was an error.

Search

The search function has two fields, the first being the format field where the user can specify the data format they prefer to receive the search results in. The available options are JSON, XML, or TEXT. The second field is the search field, where the user can enter the type of value they are searching for. The available options in this field are Title, Year, Director, Stars, Genre, and Rating. The search function uses these two fields to retrieve search results that match the user's query criteria in the specified format.

The search input is dynamic and will update the table live as you type. This searches for what you type across the whole of the database rather than a specific field. The intended use here is before hitting search and narrowing your search down to a field this will show you every instance your search appears in case you do not know which field exactly to search for. The data returned can be too much in some searches so please use the search button to narrow your search for cleaner results.

Body

The body is where the details of the films are shown in a table with features for the user to interact with the table.

Table

The table displays a list of film entries, each row representing a separate film, with columns for title, director, genre, year of release, and rating. In addition, each row contains two buttons: an "Update" button and a "Delete" button. The "Update" button allows the user to modify the details of a particular film, while the "Delete" button allows them to remove the film from the list entirely.

Update

The update button will take the user to another page where they are will be able to modify the details of a particular film

Delete

The delete button allows a user to remove the film from the list entirely. The user will be prompted to confirm they want to remove the film.

The page allows users to fill out a form to add a new film to the database. The form may contain fields such as title, director, genre, year of release, rating, and review, with each field requiring input from the user. Once the user has entered all the required information, they can submit the form to add the film to the database. The page may include features such as error checking to ensure that the user has entered all necessary information, as well as options to cancel or reset the form. Update Film Page

The page allows the user to update a film by filling out a form with the film's current details pre-filled. The form will have fields for the film's title, director, genre, year of release, rating, and review, which the user can update as desired. Before the form can be processed, the user must fill in each field with new information or the current information if no update is required. The page requires all fields are filled in before the form is submitted.
